BotFusion Documentation

Welcome to the official technical documentation for BotFusion. This platform allows you to manage multiple Telegram bots, aggregate user bases, and broadcast messages (text, image, video) to thousands of users simultaneously.

Our system is designed for high-throughput messaging, seamless user synchronization, and advanced AI-driven user behavior analysis.We provide some usefull API keys and tool

Getting Started

To start using BotFusion, you need a Telegram account and at least one Bot Token from BotFather.

1. Sign Up

  1. Navigate to the signup page.
  2. Enter your email, desired password, and your Telegram User ID.
  3. You can find your Telegram ID by messaging our Auth Bot (/start) You must start the bot first.

2. Verification

For security, we use Telegram-based 2FA. Upon signup, an OTP (One-Time Password) is sent to your Telegram account via our Auth Bot. Enter this code to verify your account.

Bot Management

Once logged in, the Dashboard is your mission control.

Adding a Bot

Enter your Bot Token (provided by @BotFather) into the "Add a New Bot" panel. The system will:

  • Validate the token with Telegram APIs.
  • Verify your plan limits (Free plan: max 10 bots).
  • Fetch the bot's username automatically.
Bot Ownership Required You must be the owner or administrator of the bot you are adding. The system validates ownership during critical operations like API Integration.

Importing Users

You can import existing user lists into BotFusion using file uploads.

For normal vps or server developers, the database is enough. For TBC developers, the users list will be available in bot dashboard.

  • Supported Formats: .txt, .csv, .json
  • Format: The file is parsed via regex for any string of 9-11 digits. You do not need a strict CSV format; a simple list of IDs works perfectly.
  • Processing: Files are processed in the background. You can navigate away while the "Scanning..." status is active.

Broadcasting

The core feature of BotFusion is sending mass messages to your aggregated user base.

Supported Content Types

  • Text: Plain text or HTML formatted messages.
  • Image: JPG/PNG files (Max 10MB) + Caption.
  • Video: MP4 files (Max 10MB) + Caption.
  • PIN Message: Pin a message to the chat by Bot.

Inline Buttons & Controls

You can add interactive buttons to your messages. These are rendered as Telegram Inline Keyboard buttons. Each button requires a Label and a valid URL.

Stop Broadcast

If you made a mistake, you can click the red Stop Broadcast button in the live status panel. Note that messages already sent to the Telegram API cannot be recalled.

Spam Detection vs. Behavior Detection

Many developers confuse spam detection with account automation / behavior anomaly detection. Your integration strategy dictates which method is right for you. BotFusion provides advanced Behavioral Anomaly Detection, which operates very differently from traditional filters.

Traditional Spam Detection

Spam filters focus on message content (e.g., keyword filtering, link blacklists, regex matching).
Example Rule: if message contains "free crypto" -> flag spam

Limitation: Fails entirely when attackers modify spam text slightly, use images instead of text, or coordinate across multiple groups to avoid duplicate detection.

Behavioral Anomaly Detection (BotFusion)

Our system analyzes user activity patterns over time. Instead of asking "Is this message spam?", it asks: "Is this user behaving like an automated system?"

Behavior Signal Example Attack Pattern
Burst Activity 20 messages sent across chats in 30 seconds.
Cross-group Activity Same user appearing active in 8 independent groups simultaneously.
Repetition Pattern Same structural message repeated predictably.
Empty Message Ratio Heavy media spam with no conversational text.
Time Entropy Activity occurring at perfectly unnatural, machine-like intervals.
What This System Does NOT Do This system does not analyze message meaning. It does not detect specific keywords, classify political content, or block specific links. It is designed for moderation tools, anti-raid systems, and community protection bots where automated attackers need to be identified regardless of their payload.

AutoUp API

AutoUp is our proprietary webhook-style system that allows your bots to automatically sync new users to your BotFusion dashboard in real-time. This eliminates the need for manual file uploads.

How it Works

  1. A user starts your bot on Telegram.
  2. Your bot code sends a POST request to our /autoup endpoint.
  3. We verify the request using your Connection Key.
  4. The user ID is instantly added to your database.

Sync User Endpoint

POST /autoup

Headers Required

Header Value Description
X-CONNECTION-KEY acct_... Your unique connection key from the dashboard.
Content-Type application/json Required.

Request & Response

Request Body 
{
  "bot_username": "@my_awesome_bot",
  "user_id": 123456789
}
Success Response (200 OK) 
{
  "status": "ok",
  "bot": "@my_awesome_bot",
  "user_id": 123456789
}

Example Implementation (Python)

import requests

def sync_user_to_dashboard(user_id, bot_username):
    url = "https://botfusion.onrender.com/autoup"
    headers = {
        "X-CONNECTION-KEY": "Your_Connection_Key_Here",
        "Content-Type": "application/json"
    }
    payload = {
        "bot_username": bot_username,
        "user_id": user_id
    }
    
    try:
        requests.post(url, json=payload, headers=headers, timeout=2)
    except:
        pass # Fail silently so bot doesn't crash

Behavior-Based Anomaly Detection API

This API enables developers to feed user activity logs into our machine learning model to receive a real-time behavioral risk score. It scales perfectly, allowing moderation bots to handle thousands of users without implementing complex ML internally.

Typical Integration Architecture

Telegram Group
      ↓
Moderation Bot (Your Code)
      ↓  (Records activity)
Local Event Logger / History
      ↓  (When trigger threshold met)
Anomaly Detection API (BotFusion)
      ↓  (Returns Risk Score)
Admin Alerts / Auto-Ban Logic

Integration Strategy: History & Triggers

Behavioral detection requires multiple messages. A single message cannot reveal patterns like burst activity or timing regularity. Therefore, the system relies on you to pass a brief history.

1. Event Collection

Your bot should record user activity locally. Keep the structure minimal to reduce your database overhead:

event = {
    "chat_id": -100123456,
    "timestamp": 1769401000,
    "text": "Hello everyone",
    "type": "text"
}
Privacy Considerations The model works primarily on activity patterns, not meaning. You can safely remove text fields, anonymize content, or store hashes instead of raw messages if your privacy policy requires it.

2. API Trigger Strategy & History Limits

To reduce API usage and respect rate limits, do not call the API on every single message. Call it at specific checkpoints.

  • Checkpoint Example: Call the API when a user reaches their 20th, 100th, 180th, and 250th message in your database.
  • History Limit: To prevent memory bloat, only keep the most recent ~300 events per user. Delete older events as they are no longer necessary for immediate behavioral analysis.

API Reference

https://botfusion.onrender.com/score_user

POST /score_user

Uses the same headers as the AutoUp API (Requires X-CONNECTION-KEY).

Request Body Format

{
  "user_id": 4021189931,
  "events": [
    {
      "chat_id": -100123456,
      "timestamp": 1769401000,
      "text": "Hello",
      "type": "text"
    },
    // ... up to 300 recent events or more
  ]
}

Response & Interpretation

{
  "user_id": 4021189931,
  "anomaly_score": 0.67,
  "risk_level": "HIGH",
  "confidence_band": "top_5_percent"
}
Anomaly Score Meaning Suggested Action (Developer Choice)
< 0.40 Safe None
0.40 – 0.55 Normal None
0.55 – 0.60 Suspicious Log for review
0.60 – 0.70 Very Suspicious Increase monitoring, restrict media
0.70 – 0.80 Risky Alert Admins, temporary mute
> 0.80 Extremely Risky Auto-ban, flag network

The API does not enforce moderation decisions. Your integration layer decides what action to take based on the returned score.

QR Generation API

This documentation provides the complete technical reference for using the BotFusion QR API. It covers every customizable key, payload structure, and branding logic required to generate high-performance, stylized QR codes.

https://botfusion.onrender.com/genqr

POST /genqr

Uses the same headers as our other APIs (Requires X-CONNECTION-KEY in the headers). Rate Limit: 20 requests per second per key.

Complete Parameter Reference

The API accepts these keys in either a JSON body or as form-data fields.

A. Core Configuration

Key Default Description
data Required The URL or text to be encoded in the QR.
scale 12 Pixel resolution per module. Values above 20 are auto-capped for server safety.
border 3 Width of the quiet zone (white space) modules around the QR.

B. Module & Body Styling

Key Default Options / Values
dot_style circle square, rounded, circle, diamond, vertical, horizontal
dot_scale 0.95 0.1 to 1.0. Set to 1.0 for solid modules with no gaps.
dot_density 1.0 0.1 to 1.0. Set to 1.0 for 100% data visibility.
fg_color #000000 Hex code for the foreground (if gradient is False).
bg_color #FFFFFF Hex code for the background.

C. Gradients

Key Default Description
gradient False Set True to enable color transitions. Overrides fg_color.
gradient_start #160400 Starting Hex color.
gradient_end #FF3700 Ending Hex color.
gradient_direction vertical Either vertical or horizontal.

D. Finder (Corner) Styling

Key Default Options / Values
finder_outer_style rounded square, rounded, circle
finder_eye_style square square, rounded, circle, diamond
finder_outer_color None Custom Hex for the outer frame (inherits body if None).
finder_eye_color None Custom Hex for the center eye (inherits body if None).
finder_outer_scale 1.0 Size multiplier for the outer frame.
finder_eye_scale 1.2 Size multiplier for the inner eye.

Logo Integration Logic

The API follows a strict hierarchy for determining the center logo.

  1. Direct File Upload: Send a file in a multipart request under the key logo. This has the highest priority.
  2. Telegram URL: Provide a direct download link via telegram_url. The proxy will fetch, rename, and forward the image to the engine.
  3. BotFusion Default: If neither of the above are provided, the engine embeds the default logo_transparent.png.
  4. Disabled: To generate a QR with no logo, you must explicitly set "logo_path": false.

Logo Customization Keys

  • logo_size_ratio: Default 0.15. Controls logo width relative to the QR. Max recommended is 0.20 for scanning safety.
  • logo_padding: Default 6. The amount of circular white-space padding behind the logo.

Usage Scenarios

Scenario A: Standard JSON (Telegram Link)

Use this for bot-to-bot communication where you have a file URL.

  • Header: Content-Type: application/json
{
  "data": "https://botfusion.wuaze.com",
  "telegram_url": "https://api.telegram.org/file/bot789/photos/file_1.jpg",
  "dot_style": "rounded",
  "gradient": true
}

Scenario B: Multipart Upload (Local File)

Use this if you are uploading a logo directly from a device or script.

  • Header: Content-Type: multipart/form-data
  • Field logo: (Binary file data)
  • Field data: https://google.com
  • Field dot_scale: 1.0
Effective Usage Tips
  • Max Scannability: Keep dot_density at 1.0 and dot_scale above 0.8. QRs with thin or missing dots are much harder for cameras to read.
  • Contrast is King: Ensure your bg_color and fg_color (or gradient) have high contrast. Dark colors on light backgrounds work best.
  • Logo Safety: Because the engine uses High (H) Error Correction, you can safely use a logo_size_ratio of up to 0.20 without losing data.
  • Sub-sampling: The engine automatically uses an ss_factor=2 (rendering at 2x size then downscaling), so even small scale values will look sharp and anti-aliased.

Rate Limits & Error Codes

To ensure global stability, our APIs enforce strict rate limiting based on your Connection Key.

  • Limit: 20 requests per second (Combined across /autoup, /score_user, and /genqr).
  • Behavior: If exceeded, the API returns HTTP 429 Too Many Requests. You should implement a queuing system in your bot to handle backoff.

Standard Error Codes

HTTP Code Meaning Solution
401 Unauthorized Missing, expired, or invalid X-CONNECTION-KEY.
403 Forbidden The bot username or resource does not belong to your account.
404 Not Found Endpoint incorrect or Bot/User validation failed.
429 Rate Limit You are sending more than 20 req/sec. Implement a queue in your bot.

FAQ

Why is my broadcast stuck?

Broadcasts may pause if Telegram returns rate limit errors (429). Our system automatically backs off and retries. If a task is manually "stopped" by the user, it cannot be resumed; you must create a new broadcast.

Can I delete a user from my audience?

Currently, the API only supports adding users. User deletion must be handled by resetting the bot data in the dashboard (Delete Bot -> Add Bot) and re-importing.

Is my data secure?

Yes. All API requests are authenticated via Connection Keys. User IDs are stored in an encrypted MongoDB cluster. We do not store raw Telegram messages from the Anomaly API longer than required for real-time inference.
